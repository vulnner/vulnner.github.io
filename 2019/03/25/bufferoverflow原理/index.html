bufferoverflow原理

Posted on 2019-03-25 | Edited on 2019-03-29 | In BufferOverflow

buffer overflow是一种很古老的攻击手段，但是它的危害并不因此而减小。

buffer overflow原理及演示

预备知识

演示平台我用了ubuntu。在开始演示之前，首先需要了解一些知识。

地址空间随机化

ubuntu和一些其他的Linux系统适用了地址空间随机化，就是使一个程序每次执行时，分配到的内存地址都是随机的，这样就是猜测确切的地址十分困难。我们可以通过如下命令关闭地址随机：

sudo sysctl -w kernel.randomize_va_space=0

StackGuard保护机制

GCC编译器内置了StackGuard保护机制，以此来防御buffer overflow。在开启之中保护机制的前提下，我们写的buffer overflow程序不会有效的。要关闭StackGuard机制，可以使用如下命令：

gcc -fno-stack-protector stack.c

栈不可执行

旧版本的ubuntu允许栈可执行，但是现在默认是不允许的，要是的我们在栈中的代码被执行，可以使用如下命令：

gcc -z execstack -o stack stack.c

这里解释了buffer overflow的一部分原理——把代码保存在栈中，然后执行。有过安全基础的都会知道，我们平时写的代码都保存在内存中的代码段，代码需要的数据则保存在栈中，两者是互相分离的。

不过在call_shellcode中，把一些汇编代码写到了buf中，并且通过gcc -z execstack使代码可以在栈中执行。

所以大致可以总结为，在用户的数据区，执行了代码。（也是代码和数据没有分离的锅啊）
